# Internationalization (i18n) Guide

## Overview

Jarvis Linux Assistant uses a lightweight, runtime-translatable i18n system built on JSON translation files. All user-facing strings are abstracted behind translation keys and resolved through a singleton `TranslationManager`.

## Architecture

### TranslationManager (`jarvis/i18n/translation_manager.py`)

- Singleton loaded at app startup
- Loads `en.json` as fallback (all keys must exist here)
- Supports runtime language switching without restart via `tr.set_language(lang)`
- Registered callbacks are notified on language change (`tr.on_change(callback)`)
- Dynamic variable interpolation: `t("key", count=5, name="foo")`

### Helper Function

```python
from jarvis.i18n import t, tr

# Get translated string
label = t("settings.language")

# With dynamic variables
text = t("workflows.created", name="my_workflow")

# Change language at runtime
tr.set_language("pt_BR")

# Register UI refresh callback
tr.on_change(self.retranslate_ui)
```

## Translation Files

Located in `translations/`:

| File | Language | Code |
|------|----------|------|
| `en.json` | English (fallback) | en |
| `pt_BR.json` | Portuguese (Brazil) | pt_BR |
| `es.json` | Spanish | es |
| `fr.json` | French | fr |
| `de.json` | German | de |
| `ja.json` | Japanese | ja |

## Key Naming Convention

Flat dot-notation keys organized by UI section:

```
<component>.<element>
```

Examples:
- `sidebar.dashboard` â€” Sidebar navigation item
- `settings.save` â€” Settings save button
- `onboarding.welcome_title` â€” Onboarding wizard title
- `diagnostics.ok_entries` â€” Diagnostics with `{count}` variable

## How to Add a New Language

1. Copy `translations/en.json` to `translations/xx.json`
2. Translate all values (keep keys identical)
3. Add the language option in `SettingsTab` dropdown (`main_window.py`):
   ```python
   ("Language", QComboBox(), "language", ["pt-PT", "en-US", "es-ES", "fr-FR", "de-DE", "xx-XX"]),
   ```
4. Add mapping in `_save_settings` and `app.py`:
   ```python
   lang_map = {..., "xx-XX": "xx"}
   ```

## How to Add a New Translation Key

1. Add the key to `translations/en.json`:
   ```json
   "my_component.my_key": "My translated text"
   ```
2. Add to all other language files
3. Use in code:
   ```python
   label = QLabel(t("my_component.my_key"))
   ```

## Runtime Language Switching

The `SettingsTab` language dropdown triggers `tr.set_language()` on save. Every UI component that needs dynamic retranslation registers a callback:

```python
tr.on_change(self.retranslate_ui)
```

The MainWindow's `retranslate_ui()` method updates tab labels and window title.

## Best Practices

- Always use `t("key")` instead of hardcoded strings
- Keep keys organized by component/section
- Use `{variable}` syntax for dynamic content
- Always add new keys to `en.json` first (it's the fallback)
- Test with all supported languages after adding new strings
- Never include HTML or formatting in translation values
